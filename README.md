# on1-ayx-journal

本仓库用于归档和发布多个独立 HTML 页面。这些页面以文档形式存放，不针对任何单个域名或具体网站，仅作为静态页面的集合。

## 项目简介

`on1-ayx-journal` 是一个通用型归档仓库，用于存放各类独立 HTML 页面。仓库本身不限定页面用途或内容方向，仅提供基本的文件组织与版本管理功能。

## 目录说明

```
.
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的资源文件（如样式、脚本、图片等）
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件为一个独立页面，文件名即页面标识。
- `assets/`：按需存放页面所需的公共资源，建议子目录按页面或类型组织。

## 页面归档说明

- 所有页面均为纯静态 HTML，无需后端支持。
- 页面之间相互独立，无依赖关系。
- 归档时请保持页面完整性，确保相关资源文件一并提交。
- 页面文件名建议使用英文小写字母、数字和连字符，避免特殊字符。

## 维护说明

- 欢迎提交新页面，请放入 `pages/` 目录并更新相关资源。
- 修改已有页面时，请保持向后兼容，或注明版本变更。
- 删除页面需确认无其他引用，并移除对应资源文件。
- 不建议在页面中嵌入外部不可控资源，以保证归档的稳定性。

## 使用方式

可直接通过浏览器打开 `pages/` 目录下的 HTML 文件查看内容，或部署至任意静态托管服务。
